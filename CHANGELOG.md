## [2.0.0] - 2026-06-21

A second full hardening cycle plus the durability and API layers that make the
running service deliver Attestia's headline promise. Every finding below was
generated by parallel audit agents and independently verified by a cross-family
review (different model families from the author) before a fix was written; each
fix shipped with a regression test proven to fail against the old code first.

### Breaking

- **Merkle tree uses RFC 6962 odd-node promotion instead of self-duplication.**
  The previous tree duplicated an unpaired node, so a tree over `[a,b,c]` and one
  over `[a,b,c,c]` produced the **same root** (CVE-2012-2459 class) — letting a
  valid inclusion proof be minted for a phantom leaf against a genuine published
  root. The tree now promotes an unpaired node unchanged, so root values are
  unambiguous. **Merkle root values change**; any attestation proof minted under
  v1.x must be re-generated. This is the only breaking change.

### Security & correctness (cycle-2)

- **Production fails closed.** The node service refused to boot in production
  with no API keys and no JWT secret (previously it silently ran with synthetic
  admin auth and a client-supplied tenant).
- **Reconciliation no longer hides cross-currency outflows.** A debit in a
  different currency than the intent was dropped from the match total *and*
  marked reconciled; it is now flagged `CURRENCY_MISMATCH` and never silently
  swallowed. Decimal-scale mismatches normalize correctly across all three
  matchers; one malformed chain amount no longer aborts the whole batch.
- **Event-store tamper-evidence hardened.** Head-truncation can no longer hide
  behind a prepended unhashed line (genesis anchor enforced once any hashed
  event exists, plus globalPosition contiguity); cross-process stale-cache
  appends fail closed instead of forking the chain; a crashed writer's stale
  lock is now broken by PID-liveness instead of deadlocking all future appends.
- **Witness & consensus.** Multi-sig idempotency checks the combined multisign
  blob hash (lost-but-applied recovery now works); the signing payload + policy
  id bind per-signer weight and public key (closes a quorum-downgrade); public
  verification consensus counts *distinct* verifier identities and requires a
  shared bundle hash (was Sybil-forgeable).
- **Chains.** XRPL partial payments report `delivered_amount`, not the requested
  ceiling; EVM transfer de-duplication keys on `(txHash, logIndex)` so distinct
  transfers in one transaction are no longer dropped.
- **Money on the execution path.** Fixed distributions no longer round-trip
  through a JS number (precision loss above 2^53); distribution and payroll
  execution are atomic (no partial-commit / wedged-plan on a zero/negative
  payout).
- **Attestation binding.** An attestation's `registry_hash` is derived from /
  asserted against the snapshot it attests, and a new `verifyAttestationBinding`
  helper closes the constitution-drift gap for external verifiers.
- **API hardening.** Constant-time JWT comparison; bounded rate-limit and
  idempotency stores; request body-size limits; metrics auth in production;
  the funding dual-control binds approver identity to the authenticated caller.

### Added — Durability

- **The running service is now durable** (opt-in via `persistence.dataDir`).
  A durable hash-chained event log is the append-only audit record; subsystem
  snapshots are the restore path; state rehydrates on restart, per-tenant, with
  a crash-window warning when the log and snapshot diverge. Default remains
  in-memory, so existing embeddings are unaffected.

### Added — API, SDK & docs

- **Treasury, Vault, and Governance are now exposed over REST** (26 new
  authenticated endpoints: payroll, distributions, funding gates, budget
  envelopes, portfolio, multi-sig signer/quorum administration) behind the same
  auth, permission, tenant, idempotency, and body-limit middleware.
- **OpenAPI 3.1** document for the full authenticated `/api/v1` surface, with a
  test that fails if any mounted route is undocumented.
- **SDK** gains `treasury`, `vault`, and `governance` namespaces plus a
  `paginateAll` auto-iterator.

### Proactive & humanization

- Unified retry/backoff across EVM, XRPL, and Solana observers; per-attempt
  submit deadlines; graceful shutdown that drains and disposes resources;
  structured telemetry on the verify/proof/persistence/governance paths;
  operator-facing errors that carry a stable code, a clear message, and an
  actionable recovery hint.

### Known limitations (roadmap)

- Single-writer-per-tenant (multi-instance HA is future work); registrar restore
  rehydrates the current frontier, with deep-lineage replay from the durable log
  on the roadmap; event-log retention/compaction is roadmap; the funding
  gatekeeper allowlist is still a fixed pair pending an admin-managed identity
  set; in persistent mode a rejected mutation may leave a spurious audit event
  (validate-then-append refinement is roadmap).

Tests: 2,220 → **2,564**, all green.

## [1.0.1] - 2026-05-29

A full health-and-hardening pass plus a new observability and error-UX feature layer. No public API was removed; existing behavior is preserved and now better instrumented and explained.

### Security & correctness hardening

Resolved 6 CRITICAL, 14 HIGH, and 19 MEDIUM findings across 14 packages. Highlights:

- **Append-only lineage re-architecture** — lineage tracking reworked so that history is structurally unbroken and tamper-evident, not merely conventionally append-only.
- **Runtime dual-witness enforcement** — dual-witness validation is now enforced at runtime on critical operations, not only asserted at the type level.
- **Attestation-proof leaf binding** — Merkle inclusion proofs are bound to their attestation leaf, closing a proof-substitution gap.
- **Hash-chain genesis anchor** — the event hash chain is anchored to an explicit genesis marker so the first event cannot be silently replaced.
- **Tenant-scoped idempotency** — `Idempotency-Key` handling is scoped per tenant, preventing cross-tenant key collisions.
- **Verify-then-count multi-sig** — multi-sig quorum counts signatures only after verifying each one, preventing inflated quorum via unverified signatures.

### Added — Observability layer

- **Injectable telemetry** — domain packages accept an optional `Telemetry` sink (defaulting to a silent no-op), so verification and accounting run unobserved and free unless a host opts in. Telemetry is defensively guarded: a throwing sink can never break the operation it observes.
- **pino + Prometheus bridge** — the node service wires the telemetry contract to structured `pino` logging and Prometheus counters/histograms, including `attestia_witness_total{status}`.

### Added — Error-UX layer

- **`{ code, message, hint }` error envelope** — API errors carry a stable machine code, a safe human-readable message, and an actionable hint. Internal details never leak in 4xx responses.
- **Structured reconciliation discrepancies** — reconciliation responses now return discrepancies in a structured, machine-readable shape rather than free text.
- **Auditor report formatters** — verifier reports gain human-readable formatters for auditor consumption.
- **SDK typed errors** — the SDK surfaces typed errors with `hint` and `ValidationIssue` details for consumers.

### Fixed — Documentation accuracy

- Test counts updated from 1,928 to **2,220** across README, HANDBOOK, INDEX, the landing-page config, and the docs handbook.
- `VERIFICATION_GUIDE.md`: corrected the Step 4 replay example (auditors are pointed at `runVerification()`, which recomputes the global hash from replayed snapshots plus the bundle's own chain hashes; the prior snippet passed the global hash as `expectedHash` to `verifyByReplay()`, producing a spurious FAIL on any chain-observing node). Stale "1,107 tests" corrected to 2,220.
- Documented Node floor reconciled to **Node 20+** (CI tests Node 20 and 22) across VERIFICATION_GUIDE and HANDBOOK.
- HANDBOOK version corrected to 1.0.1; REST endpoint count stated as 34; event-type count reconciled to 34 (verified against the event-store catalog).
- INDEX workflow inventory corrected (3 workflows: ci, docker, pages).

### Security

- All dependency advisories cleared — **0 high/critical** advisories.
