## मिशन

हमारा मानना ​​है कि धन - चाहे वह कहीं भी हो, और कैसे भी आगे बढ़े - उसी कठोरता का हकदार है जो इसे बनाने वाली प्रणालियों में है। स्मार्ट अनुबंध निष्पादित होते हैं। ब्लॉकचेन रिकॉर्ड करते हैं। लेकिन कोई भी *पुष्टि* नहीं करता।

एटटेस्टिया वह लापता परत है: संरचनात्मक शासन, नियतात्मक लेखांकन और मानव-अनुमोदित इरादा - जो श्रृंखलाओं, संगठनों और व्यक्तियों में एकीकृत है।

हम आपके धन को स्थानांतरित नहीं करते हैं। हम यह साबित करते हैं कि क्या हुआ, यह सीमित करते हैं कि क्या हो सकता है, और वित्तीय रिकॉर्ड को अटूट बनाते हैं।

### हम किस चीज के लिए खड़े हैं

- **गति से अधिक सत्य।** प्रत्येक वित्तीय घटना केवल एक बार जोड़ी जा सकती है, इसे दोहराया जा सकता है, और इसका समाधान किया जा सकता है। यदि इसे साबित नहीं किया जा सकता है, तो यह नहीं हुआ।
- **मनुष्य अनुमोदित करते हैं; मशीनें सत्यापित करती हैं।** एआई सलाह देता है, स्मार्ट अनुबंध निष्पादित होते हैं, लेकिन बिना स्पष्ट मानव प्राधिकरण के कुछ भी आगे नहीं बढ़ता। कभी नहीं।
- **संरचनात्मक शासन, न कि राजनीतिक शासन।** हम इस पर मतदान नहीं करते हैं कि क्या मान्य है। हम अपरिवर्तनीय नियमों को परिभाषित करते हैं जो बिना शर्त लागू होते हैं - पहचान स्पष्ट है, वंशावली अखंड है, क्रम नियतात्मक है।
- **इरादा निष्पादन नहीं है।** आप जो चाहते हैं उसे घोषित करना और उसे करना अलग-अलग कार्य हैं जिनमें अलग-अलग द्वार हैं। उनके बीच का अंतर वह जगह है जहाँ विश्वास मौजूद होता है।
- **श्रृंखलाएँ गवाह हैं, प्राधिकरण नहीं।** एक्सआरपीएल पुष्टि करता है। एथेरियम निपटान करता है। लेकिन अधिकार किसी भी श्रृंखला की सहमति से नहीं, बल्कि संरचनात्मक नियमों से प्रवाहित होता है।
- **उबाऊ बुनियादी ढांचा जीतता है।** दुनिया को एक और डीफाई प्रोटोकॉल की आवश्यकता नहीं है। इसे इसके नीचे लेखांकन परत की आवश्यकता है - वह वित्तीय प्रणाली जो हर चीज को विश्वसनीय बनाती है।

## इसे 2 मिनट में आजमाएं

एटटेस्टिया को समझने का सबसे तेज़ तरीका है कि एक भुगतान कैसे होता है, यह पूरी तरह से देखें। इंटरैक्टिव डेमो **पूर्ण इरादा → अनुमोदन → निष्पादन → सत्यापन → पुष्टि → प्रमाण** पाइपलाइन को अंत-से-अंत तक चलाता है - प्रत्येक चरण वास्तविक डोमेन पैकेजों (मिलान, हैशिंग, एक्सआरपीएल-शैली की पुष्टि, मर्कल प्रूफ) के खिलाफ वास्तविक रूप से गणना की जाती है, न कि नकली।

```bash
pnpm install   # Install all dependencies
pnpm build     # Build all packages
pnpm demo      # Walk the full pipeline (~10s, paced for readability)
```

आप देखेंगे कि एक एकल वेतन भुगतान कैसे स्वतंत्र रूप से सत्यापित क्रिप्टोग्राफ़िक प्रमाण में बदल जाता है, चरण दर चरण। गति को छोड़ने और तुरंत चलाने के लिए `--fast` जोड़ें: `pnpm demo --fast` (`pnpm demo --help` सभी झंडे सूचीबद्ध करता है)।

## मुख्य पैटर्न

प्रत्येक इंटरैक्शन एक प्रवाह का पालन करता है:

```
Intent → Approve → Execute → Verify
```

1. **इरादा** - एक उपयोगकर्ता या सिस्टम वांछित परिणाम घोषित करता है
2. **अनुमोदन** - रजिस्ट्रम संरचनात्मक रूप से मान्य करता है; एक मानव स्पष्ट रूप से हस्ताक्षर करता है
3. **निष्पादन** - ऑन-चेन लेनदेन सबमिट किया जाता है
4. **सत्यापन** - समाधान पुष्टि करता है; एक्सआरपीएल रिकॉर्ड की पुष्टि करता है

कोई भी चरण वैकल्पिक नहीं है। कोई भी चरण स्वचालित रूप से हटाया नहीं गया है।

### एक्सआरपीएल एकीकरण परीक्षण

एक स्टैंडअलोन `रिप्ड` नोड डॉकर में चलता है ताकि नियतात्मक ऑन-चेन एकीकरण परीक्षण किया जा सके - कोई टेस्टनेट निर्भरता नहीं, कोई फ़ॉसेट नहीं, सब-सेकंड लेज़र क्लोज।

```bash
docker compose up -d              # Start standalone rippled
pnpm --filter @attestia/witness run test:integration  # Run on-chain round-trip tests
docker compose down               # Stop rippled
```

## सुरक्षा और डेटा का दायरा

- **पहुंची गई डेटा:** वित्तीय लेज़र प्रविष्टियों, सत्यापन रिकॉर्ड और क्रिप्टोग्राफ़िक प्रमाणों को पढ़ता और लिखता है। जब साक्षी मॉड्यूल सक्रिय होता है तो ब्लॉकचेन नोड्स (एक्सआरपीएल) से जुड़ता है।
- **पहुंची नहीं गई डेटा:** कोई टेलीमेट्री नहीं। कोई उपयोगकर्ता क्रेडेंशियल संग्रहण नहीं। कोई तृतीय-पक्ष विश्लेषण नहीं।
- **आवश्यक अनुमतियाँ:** स्थानीय डेटा निर्देशिकाओं तक पढ़ने/लिखने की पहुँच। केवल ब्लॉकचेन सत्यापन के लिए नेटवर्क एक्सेस। पूर्ण स्ट्राइड विश्लेषण के लिए [THREAT_MODEL.md](THREAT_MODEL.md) देखें।
