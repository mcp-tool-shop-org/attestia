## मिशन

हमारा मानना है कि पैसा - चाहे वह कहीं भी हो, चाहे वह किसी भी तरह से गति करे - उन प्रणालियों के समान कठोरता का हकदार है जिन्होंने इसे बनाया है। स्मार्ट अनुबंध निष्पादित होते हैं। ब्लॉकचेन रिकॉर्ड करते हैं। लेकिन कोई *प्रमाणित* नहीं करता।

एटिस्टिया एक महत्वपूर्ण परत है: संरचनात्मक शासन, नियतात्मक लेखांकन और मानव-अनुमोदित इरादा - जो सभी श्रृंखलाओं, संगठनों और व्यक्तियों में एकीकृत हैं।

हम आपके पैसे को नहीं हिलाते हैं। हम यह साबित करते हैं कि क्या हुआ, यह सीमित करते हैं कि क्या हो सकता है, और वित्तीय रिकॉर्ड को अविनाशी बनाते हैं।

### हम किस बात के लिए खड़े हैं

- **सत्य, गति से ऊपर।** प्रत्येक वित्तीय घटना अपरिवर्तनीय है, पुन: चलाने योग्य है और मिलान योग्य है। यदि इसे साबित नहीं किया जा सकता है, तो यह नहीं हुआ।
- **मनुष्य अनुमोदन करते हैं; मशीनें सत्यापित करती हैं।** एआई सलाह देता है, स्मार्ट अनुबंध निष्पादित करते हैं, लेकिन कुछ भी बिना स्पष्ट मानव प्राधिकरण के नहीं होता। कभी नहीं।
- **संरचनात्मक शासन, राजनीतिक शासन नहीं।** हम यह नहीं तय करते कि क्या मान्य है। हम अपरिवर्तनीय नियमों को परिभाषित करते हैं जो बिना शर्त लागू होते हैं - पहचान स्पष्ट है, विरासत अटूट है, क्रम नियतात्मक है।
- **इरादा निष्पादन नहीं है।** आप जो चाहते हैं उसे घोषित करना और उसे करना दो अलग-अलग कार्य हैं, जिनके अलग-अलग चरण हैं। उनके बीच का अंतर वह जगह है जहां विश्वास रहता है।
- **श्रृंखलाएं गवाह हैं, प्राधिकरण नहीं।** XRPL प्रमाणित करता है। एथेरियम निपटान करता है। लेकिन प्राधिकरण संरचनात्मक नियमों से आता है, न कि किसी भी श्रृंखला के आम सहमति से।
- **सरल बुनियादी ढांचा ही जीतता है।** दुनिया को एक और DeFi प्रोटोकॉल की आवश्यकता नहीं है। इसे नीचे की लेखांकन परत की आवश्यकता है - वह वित्तीय बुनियादी ढांचा जो बाकी सब कुछ विश्वसनीय बनाता है।

## इसे 2 मिनट में आजमाएं

एटिस्टिया को समझने का सबसे तेज़ तरीका एक भुगतान प्रवाह को पूरी तरह से देखना है। इंटरैक्टिव डेमो **इरादा → अनुमोदन → निष्पादन → सत्यापन → प्रमाणीकरण → प्रमाण** पाइपलाइन को अंत से अंत तक चलाता है - प्रत्येक चरण वास्तविक डोमेन पैकेजों (मेलिंग, हैशिंग, XRPL-शैली प्रमाणीकरण, मर्केल प्रमाण) के खिलाफ वास्तविक रूप से गणना किया जाता है, न कि एक मॉक।

```bash
pnpm install   # Install all dependencies
pnpm build     # Build all packages
pnpm demo      # Walk the full pipeline (~10s, paced for readability)
```

आप देखेंगे कि एक एकल वेतन भुगतान एक स्वतंत्र रूप से सत्यापित क्रिप्टोग्राफिक प्रमाण में बदल जाता है, चरण दर चरण। गति को छोड़ने और तुरंत चलाने के लिए `--fast` जोड़ें: `pnpm demo --fast` (`pnpm demo --help` सभी ध्वजों को सूचीबद्ध करता है)।

## मुख्य पैटर्न

प्रत्येक इंटरैक्शन एक प्रवाह का पालन करता है:

```
Intent → Approve → Execute → Verify
```

1. **इरादा** - एक उपयोगकर्ता या सिस्टम एक वांछित परिणाम घोषित करता है
2. **अनुमोदन** - रजिस्ट्रम संरचनात्मक रूप से मान्य करता है; एक मानव स्पष्ट रूप से हस्ताक्षर करता है
3. **निष्पादन** - ऑन-चेन लेनदेन सबमिट किया जाता है
4. **सत्यापन** - मिलान पुष्टि करता है; XRPL रिकॉर्ड को प्रमाणित करता है

कोई भी चरण वैकल्पिक नहीं है। कोई भी चरण स्वचालित नहीं है।

### एक्सआरपीएल एकीकरण परीक्षण

एक स्टैंडअलोन `रिप्ल्ड` नोड डॉकर में चलता है, जो नियतात्मक ऑन-चेन एकीकरण परीक्षणों के लिए है — किसी भी टेस्टनेट पर निर्भरता नहीं, कोई फ़ॉसेट नहीं, उप-सेकंड में लेज़र बंद।

```bash
docker compose up -d              # Start standalone rippled
pnpm --filter @attestia/witness run test:integration  # Run on-chain round-trip tests
docker compose down               # Stop rippled
```

## सुरक्षा और डेटा दायरा

- **आवश्यक अनुमतियाँ:** स्थानीय डेटा निर्देशिकाओं तक पढ़ने/लिखने की पहुंच। केवल ब्लॉकचेन प्रमाणीकरण के लिए नेटवर्क एक्सेस। पूर्ण STRIDE विश्लेषण के लिए [THREAT_MODEL.md](THREAT_MODEL.md) देखें।
